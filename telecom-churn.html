Telecom Churn Prediction Service

Machine Learning FastAPI Logistic Regression Full-Stack

1. The Challenge

Customer retention is a critical metric for telecommunications companies. The cost of acquiring a new customer is significantly higher than retaining an existing one. The objective of this project was to build a machine learning pipeline capable of identifying customers with a high probability of canceling their service, allowing the business to deploy targeted retention strategies before the churn occurs.

2. The Solution & Architecture

I developed a full-stack machine learning application that serves predictions in real-time. The architecture is split into a robust backend API and a responsive frontend interface.

🧠 Machine Learning

Tested multiple algorithms including Random Forest, LightGBM, and CatBoost. Selected Logistic Regression as the final model due to its strong performance (0.75 ROC-AUC) and excellent interpretability for business stakeholders.

⚙️ Backend API

Deployed the serialized model using a FastAPI REST API hosted on Render. It handles incoming JSON payloads, processes the features, and returns probability scores asynchronously.

3. Data-Driven Results

To align the model with actual business goals, I optimized the classification threshold using the Precision-Recall curve.

  • Threshold Optimization: Maximized the F1-score at a custom threshold of 0.478 instead of the default 0.5.
  • Risk Mitigation: This adjustment significantly reduced false negatives (down to 79 cases), ensuring the business rarely misses a customer at risk of leaving.
  • Dynamic UI: The frontend visually categorizes users into risk tiers (Low, Medium, High) with reactive progress bars based on the API's live response.
